# MANIFEST – tof-bridge-planning-method

## Purpose

This bridge-planning baseline continues prepared building blocks into rough sketches and target candidates while preserving openness and preventing silent runtime overclaim.

## Required chain

1. read prepared building blocks
2. derive rough sketches
3. derive target candidates
4. report open directions and review requirements
5. keep planning artifacts separate from runtime truth

## Stable candidate classes

- discord_candidate
- bot_candidate
- repo_candidate
- review_required

## Not allowed

- silently convert planning into runtime truth
- claim target candidates are implementation
- erase uncertainty
- force one direction where evidence is still open
- materialize target code in the default run
